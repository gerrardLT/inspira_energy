## F1 — 基金产品卡片化展示

### TS-01：基金卡片网格布局（对应 US-01）

**测试目标：** 验证四类基金卡片在各设备尺寸下正确渲染并显示合规标注

**起始条件：** 官网已部署，基金页面可访问，四类基金数据已在后台配置

**用户角色：** 机构投资人

**操作步骤：**
1. 在桌面浏览器（≥1200px）访问 `/funds` → 四张卡片以 4 列网格显示，间距一致
2. 缩小至平板尺寸（768px）→ 卡片变为 2 列布局
3. 缩小至手机尺寸（375px）→ 卡片变为单列堆叠
4. 检查每张卡片 → 均显示：基金名称、描述、收益区间、风险等级、地区标签、"了解更多"按钮
5. 检查收益区间文字下方 → 显示合规标注"*非承诺收益，仅供参考"

**预期结果：**
- 三种尺寸布局均符合设计规范，无布局崩坏
- 合规标注在所有尺寸下可见且文字完整
- 页面加载时间 < 3 秒（Lighthouse 验证）

**边界情况：**
- 某类基金数据缺失时，该卡片显示"即将推出"占位符，不报错

### TS-02：基金详情抽屉交互（对应 US-02）

**测试目标：** 验证抽屉面板打开/关闭行为和内容完整性

**起始条件：** 基金页面已加载

**用户角色：** 机构投资人

**操作步骤：**
1. 点击任一基金卡片的"了解更多" → 右侧抽屉滑出，展示详情内容
2. 检查抽屉内容 → 包含：基金描述、资金流向说明、适合投资人类型、最低投资额、锁定期、"提交投资意向"按钮
3. 点击遮罩层（抽屉外区域）→ 抽屉关闭
4. 重新打开抽屉，按 ESC 键 → 抽屉关闭
5. 重新打开抽屉，点击右上角关闭按钮 → 抽屉关闭
6. 打开抽屉，点击内部"提交投资意向"按钮 → 跳转至意向表单页面

**预期结果：**
- 抽屉打开/关闭动画流畅（≤300ms）
- 抽屉内容随当前语言切换（中英文）
- 关闭后焦点回到触发按钮

## F2 — LP 投资意向预登记表单

### TS-04：表单必填字段验证（对应 US-04）

**测试目标：** 验证表单验证逻辑和错误提示

**起始条件：** 投资意向表单页面已加载

**用户角色：** 机构投资人

**操作步骤：**
1. 不填写任何字段，直接点击"提交" → 姓名、机构、邮箱字段显示红色错误提示
2. 填写姓名但邮箱格式错误（如"abc"）→ 邮箱字段显示"请输入有效邮箱"
3. 填写正确邮箱但未选基金类型 → 基金类型字段显示错误提示
4. 全部必填字段正确填写后点击提交 → 表单提交成功

**预期结果：**
- 错误提示在对应字段下方，不遮挡其他内容
- 所有必填字段校验通过后才能提交
- 提交后页面跳转至感谢页面

**边界情况：**
- 同一邮箱在 24 小时内重复提交 → 显示"您已提交过"提示，数据不重复写入

### TS-05：预填基金类型跳转（对应 US-05）

**测试目标：** 验证从基金详情跳转时基金类型正确预填

**起始条件：** 用户在基金详情抽屉页

**用户角色：** 机构投资人

**操作步骤：**
1. 打开"债权基金"详情抽屉，点击"提交投资意向" → 跳转至表单页，基金类型"债权基金"已选中
2. 检查 URL → 包含 `?fund_type=debt` 参数
3. 手动修改基金类型为"股权基金" → 字段可正常修改
4. 直接访问 `/invest?fund_type=cod` → COD 基金已预填选中

**预期结果：**
- 预填字段与触发来源一致
- 预填字段可被用户修改
- 其他字段保持空白

**测试目标：** 验证提交后 Supabase 写入和 SendGrid 邮件通知均正确触发

**起始条件：** 测试邮箱已配置，SendGrid API 可用

**预期结果：**
- 数据写入 Supabase 延迟 < 3 秒

## F4 — 路条信息收集表单

### TS-09：分步表单流程（对应 US-08）

**测试目标：** 验证三步表单流程的正确性和数据持久性

**起始条件：** 路条提交页面已加载

**用户角色：** 新能源开发商

**操作步骤：**
1. Step 1：填写公司名称、联系人、邮箱 → 点击"下一步" → 进入 Step 2，进度条更新
2. Step 2：邮箱格式错误，点击"下一步" → 邮箱字段显示错误，停留在 Step 2
3. 修正邮箱，选择地区和项目类型 → 点击"下一步" → 进入 Step 3
4. 点击"上一步" → 回到 Step 2，之前填写内容保留
5. Step 3：填写阶段和投建时间 → 点击"提交" → 显示确认页面

**预期结果：**
- 进度条实时更新（1/3、2/3、3/3）
- 回退时已填数据不丢失
- 提交后数据写入 Supabase，文件（如有）写入 Storage

**边界情况：**
- 容量字段输入负数或 0 → 显示"请输入有效容量"错误
- 浏览器刷新后 → 已填写步骤数据丢失（可接受，需重新填写）

---

### TS-10：文件上传（对应 US-09）

**测试目标：** 验证文件上传功能、格式限制和错误处理

**起始条件：** 用户在 Step 3 页面

**用户角色：** 新能源开发商

**操作步骤：**
1. 拖拽上传一个 PDF 文件（5MB）→ 上传进度条显示，完成后显示文件名
2. 上传一个 15MB 文件 → 显示"文件大小超过 10MB 限制"错误
3. 上传一个 .exe 文件 → 显示"不支持的文件格式"错误
4. 连续上传 6 个文件 → 第 6 个文件被拒绝，提示"最多上传 5 个文件"
5. 点击已上传文件的删除按钮 → 文件从列表移除
6. 不上传任何文件直接提交 → 提交成功（文件上传为可选）

**预期结果：**
- 支持格式：PDF、DOC、DOCX、JPG、PNG
- 文件成功上传至 Supabase Storage 私有 bucket
- 上传失败时显示重试按钮，点击重试可恢复

## F7 — 中英双语切换

### TS-16：语言切换功能（对应 US-15）

**测试目标：** 验证语言切换正确性、持久性和页面保持

**起始条件：** 官网任意页面已加载，默认英文

**用户角色：** 通用用户

**操作步骤：**
1. 点击右上角"中"按钮 → 0.5 秒内切换为中文，所有内容更新
2. 刷新页面 → 仍为中文（localStorage 持久化）
3. 切换语言时观察 URL → 参数变为 `?lang=zh`，页面不跳转
4. 点击导航栏各页面 → 均显示中文内容
5. 在新标签页打开官网 → 默认显示英文（新浏览器无 localStorage）

**预期结果：**
- 切换速度 < 500ms
- 所有页面内容均有中英文版本，无遗漏
- URL 参数正确更新

**边界情况：**
- 手动修改 URL 参数为 `?lang=fr`（不支持的语言）→ 回退显示英文
