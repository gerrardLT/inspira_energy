# Sprint Plan — Inspira Energy 官网（一期）

> 目标：6 周内完成 MVP 上线
> 用户故事：18 个（`docs/User-Stories-Inspira-Energy-Website.md`）
> 测试场景：19 个（`docs/Test-Scenarios-Inspira-Energy-Website.md`）
> 技术栈：Next.js + Tailwind + Mapbox + D3.js + Supabase + SendGrid

---

## 团队容量

| 角色 | 人数 | 周期 | 每周可用 |
|------|------|------|---------|
| 前端开发 | 1 | 全职 | 40h |
| 后端/全栈开发 | 1 | 全职 | 40h |
| 设计师 | 1 | 兼职（Sprint 1-2） | 20h |

**每 Sprint 容量：** ~20 story points（含 15% buffer 应对意外）
**Sprint 周期：** 1 周

---

| — | 项目初始化（Next.js + Tailwind + Supabase + Git） | 2 | 后端 | — |

### 定义完成（DoD）
- [ ] 项目可在本地 `npm run dev` 启动

| 风险 | 影响 | 缓解措施 |

## Sprint 2（Week 2）：表单系统 + Supabase 后端

**Sprint Goal：** 完成两个核心表单及其后端数据存储，建立完整的数据提交链路。

### 故事列表

| 故事 | 标题 | Points | 负责人 | 依赖 |
|------|------|--------|--------|------|
| US-04 | LP 投资意向表单（含验证） | 5 | 全栈 | Supabase schema |
| US-05 | 基金详情跳转预填 | 3 | 前端 | US-02, US-04 |
| US-08 | 路条分步表单 | 5 | 全栈 | Supabase schema |
| US-09 | 路条文件上传 | 5 | 后端 | Supabase Storage |
| — | 设计稿交付（表单/地图/双语） | 2 | 设计师 | — |

**Total Points：** 20（含 buffer 约 17 实际）

### 定义完成（DoD）
- [ ] 两类表单均可完整提交，数据写入 Supabase 对应表
- [ ] 必填字段验证正确，错误提示显示在字段下方
- [ ] 文件上传成功至 Supabase Storage，文件大小限制生效
- [ ] 从基金详情跳转表单，基金类型字段正确预填

### 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Supabase Storage 权限配置复杂 | 文件上传无法正常工作 | 后端提前在 Sprint 1 完成 Storage bucket 配置 |
| 路条表单字段较多，用户弃填率高 | 影响开发商转化 | 上线后通过 Hotjar 观察弃填节点，优化 |

## Sprint 3（Week 3）：数据可视化（地图 + 基金结构）

**Sprint Goal：** 完成两个高视觉冲击力的数据可视化功能，提升官网专业感和信任度。

### 故事列表

| 故事 | 标题 | Points | 负责人 | 依赖 |
|------|------|--------|--------|------|
| US-11 | 全球市场地图渲染 | 5 | 前端 | Mapbox API Key |
| US-12 | 地图地区详情卡片 | 3 | 前端 | US-11, Supabase 数据 |
| US-06 | 基金结构动态图解 | 8 | 前端 | F1 页面结构, D3.js |
| US-07 | 风险收益曲线交互 | 5 | 前端 | US-06 |
| US-03 | 地区筛选功能 | 3 | 前端 | US-01 |

**Total Points：** 24（高复杂度 Sprint，需密切协作）

### 定义完成（DoD）
- [ ] 地图在 2 秒内完成渲染，四个地区光点颜色区分正确
- [ ] 点击光点弹出详情卡片，移动端底部抽屉形式
- [ ] 基金结构图解 Tab 切换响应 < 500ms，动画流畅（≥30fps）
- [ ] 地区筛选正确过滤卡片，有 fade 过渡动画

### 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| D3.js 复杂度高，开发时间超预期 | 压缩测试时间 | 优先保证静态版本，动态交互作为增强 |
| Mapbox API Key 申请延迟 | 地图无法渲染 | 准备 Leaflet 作为备选方案 |
| 本 Sprint 点数偏高（24） | 无法按时完成 | US-07 风险收益曲线可降级至 Sprint 4 |

## Sprint 4（Week 4）：国际化 + 邮件系统

**Sprint Goal：** 完成中英文双语支持和邮件通知链路，功能开发收尾。

### 故事列表

| 故事 | 标题 | Points | 负责人 | 依赖 |
|------|------|--------|--------|------|
| US-15 | 全局语言切换功能 | 5 | 全栈 | 所有页面完成 |
| US-16 | 表单字段双语一致性 | 3 | 前端 | US-15, 表单完成 |
| US-06 | 提交意向后团队邮件通知 | 3 | 后端 | US-04, SendGrid |
| US-10 | 路条提交后双方邮件通知 | 3 | 后端 | US-08, SendGrid |
| US-17 | 邮件订阅功能 | 3 | 全栈 | Supabase + SendGrid |
| US-18 | 订阅确认邮件 | 2 | 后端 | US-17, SendGrid |

**Total Points：** 19

### 定义完成（DoD）
- [ ] 所有页面中英文切换 < 500ms，内容无遗漏
- [ ] 语言选择通过 localStorage 持久化，刷新后保持
- [ ] 三类邮件（团队通知、开发商确认、订阅确认）均在 5 分钟内到达
- [ ] 同一邮箱重复订阅显示提示，不重复写入

### 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 双语翻译工作量超预期 | 内容不完整 | 英文优先，中文 Sprint 5 补充完善 |
| SendGrid 账户配置延迟 | 邮件功能无法测试 | Sprint 2 提前申请账户，Sprint 3 完成配置 |

## Sprint 5（Week 5）：集成测试 + 内容填充 + 响应式优化

**Sprint Goal：** 完成全量集成测试，填充真实内容，确保各设备体验一致。

### 工作内容

| 任务 | Points | 负责人 | 说明 |
|------|--------|--------|------|
| 全量集成测试（TS-01 至 TS-19） | 5 | 全员 | 覆盖所有测试场景，记录 Bug |
| 真实内容填充（团队介绍/基金文案） | 3 | 创始人 + 前端 | 替换占位文字和占位图 |
| 响应式优化（三端） | 3 | 前端 | 桌面/平板/手机逐页验证 |
| SEO 基础配置（meta/sitemap） | 2 | 后端 | Google 可正常索引 |
| Google Analytics + Hotjar 接入 | 2 | 后端 | 埋点验证 |
| Bug 修复（预估 5-8 个） | 3 | 全员 | 来自测试发现的 Bug |

**Total Points：** 18

### 定义完成（DoD）
- [ ] 19 个测试场景全部执行，通过率达 90%+
- [ ] 所有 P0 Bug 已修复（表单、邮件、双语相关）
- [ ] 真实内容填充完成，无占位符遗留
- [ ] 三端响应式验证通过，无布局崩坏
- [ ] Google Analytics 数据正常上报

### 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 真实内容（文案/图片）准备延迟 | 无法完成填充 | 提前 2 周通知团队准备素材 |
| Bug 数量超预期 | 延长修复时间 | 优先修复阻断性 Bug，UI 问题记录至 Sprint 6 |

## Sprint 6（Week 6）：UAT + 上线准备

**Sprint Goal：** 完成用户验收测试、合规审查，官网正式上线。

### 工作内容

| 任务 | Points | 负责人 | 说明 |
|------|--------|--------|------|
| UAT（创始人 + 目标用户测试） | 3 | 创始人 | 邀请 2-3 位 LP/开发商走完整流程 |
| UAT Bug 修复 | 3 | 全员 | UAT 发现的问题 |
| 合规审查（基金展示表述） | 2 | 创始人 + 法务 | MAS 合规确认 |
| 性能优化（Lighthouse 达标） | 2 | 前端 | 桌面/移动 Lighthouse > 80 |
| 域名配置 + Vercel 部署 | 2 | 后端 | 正式域名绑定，SSL 证书 |
| 上线发布 + 监控配置 | 2 | 全员 | 正式发布，配置错误监控 |

**Total Points：** 14

### 上线标准（Go/No-Go）
- [ ] UAT 通过，无阻断性 Bug
- [ ] 法务合规审查通过（基金收益表述）
- [ ] Lighthouse 桌面 > 90，移动 > 80
- [ ] 表单 → Supabase → 邮件 全链路测试通过
- [ ] SSL 证书有效，HTTPS 正常
- [ ] 错误监控（Sentry 或 Vercel 内置）已配置
