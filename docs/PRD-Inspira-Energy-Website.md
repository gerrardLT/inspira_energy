# PRD — Inspira Energy 官网 + 业务平台（一期）

## 1. Summary

本文档定义 Inspira Energy 官网一期产品需求。目标是为这家新加坡新能源基金投资平台打造一个专业官网，展示基金产品、全球布局、团队背景，同时提供机构投资人意向收集和路条信息收集功能，作为首个对外可信资产。

## 3. Background

### 3.1 背景

Inspira Energy Pte Ltd 是一家注册于新加坡的新能源基金投资平台，业务覆盖中国、东南亚、欧洲和澳大利亚。公司提供四类基金产品（债权/股权/投建/COD），与开发商合作投建光伏、储能、风电电站，并开展 COD 收购持有运营。

### 3.2 为什么现在做

- 公司处于初创期，需要一个专业官网建立对外信任
- 东南亚新能源市场快速增长，GCPC Group 等竞争对手已建立在线形象
- 机构投资人和开发商都会通过官网首次接触公司，官网是 LP 漏斗的第一环
- 路条信息资源需要通过在线入口收集，建立信息壁垒

### 3.3 近期变化使这成为可能

- AI 建站工具和现代前端框架降低了建站成本
- 交互式数据可视化技术成熟，可低成本实现基金结构可视化
- 新加坡新能源行业关注度持续上升，SEO 红利期

---

## 4. Objective

### 4.1 目标

打造一个专业、可信、高转化的新能源基金投资平台官网，让投资人和开发商在首次访问时建立信任并采取行动（提交投资意向/路条信息）。

### 4.2 对齐战略

- 对齐产品愿景：*The smart capital behind clean energy — from greenfield to grid.*
- 对齐 GTM 策略：官网是关系驱动型获客的数字化放大器
- 对齐差异化：基金产品化展示 + 路条流通机制 = 竞品不具备的数字能力

### 4.3 关键结果（OKR）

| KR | 目标值 | 时间 |

| KR2 | 获得 30+ 个 LP 投资意向提交 | 上线后 90 天内 |
| KR3 | 获得 10+ 条路条信息提交 | 上线后 90 天内 |

## 5. Market Segment(s)

### 5.1 目标细分市场

| 细分市场 | 核心需求（JTBD） | 约束 |
|----------|-----------------|------|
| A. 机构投资人 | 寻找新能源资产配置渠道，了解 Inspira 的基金产品是否符合其风险偏好 | 需要高信任感、数据透明、合规背书 |
| B. 新能源开发商 | 寻找资金合作方或路条转让渠道，快速变现或引入资本 | 希望流程简单、反馈及时、信息保密 |

### 5.2 关键约束

- **信任门槛高**：新能源基金涉及大额资金，投资人不会仅凭官网就做决策，官网目标是获取初步兴趣
- **合规约束**：基金产品展示需符合新加坡 MAS 监管要求，避免承诺固定收益
- **双语需求**：面向中国开发商需要中文，面向国际 LP 需要英文

## 6. Value Proposition(s)

### 6.1 对机构投资人

| 维度 | 内容 |
|------|------|
| 核心需求 | 寻找符合 ESG 要求、收益稳定、风险可控的新能源资产配置 |
| 客户获得 | 四类标准化基金产品，覆盖从开发到运营的全周期，可按风险偏好选择 |
| 避免的痛点 | 无需自建团队直投项目，避免信息不对称和运营风险 |
| 优于竞品的地方 | GCPC Group 无基金产品化展示；Actis 只做大项目门槛高；Inspira 专注中小项目，响应更快 |

### 6.2 对新能源开发商

| 维度 | 内容 |
|------|------|
| 核心需求 | 获得项目资金或快速转让路条 |
| 客户获得 | 符合标准的项目共同投建，不符合标准的路条协助转让 |
| 避免的痛点 | 避免漫长的银行贷款审批、避免单一买家压价 |
| 优于竞品的地方 | Inspira 提供"投建 + 转让"双通道，开发商永远有退路 |

## 7. Solution

### 7.1 页面架构与用户流程

```
首页（Hero + 交互地图 + 核心价值）
├── About Us（公司介绍 + 团队背书）
├── Funds（四类基金产品卡片）
│   ├── Debt Fund（债权基金）
│   ├── Equity Fund（股权基金）
│   ├── Development Fund（投建基金）
│   └── COD Fund（COD基金）
├── Projects（项目管道 + 市场地图）
├── For Developers（开发商服务 + 路条提交）
├── Insights（市场洞察 + 邮件订阅）
└── Contact（联系方式 + 通用咨询表单）
```

**用户流程 A（投资人）：**
首页浏览 → 查看基金产品卡片 → 阅读基金结构可视化 → 提交投资意向表单

**用户流程 B（开发商）：**
首页浏览 → 进入开发商服务页 → 提交路条信息 → 收到确认邮件

### 7.2 核心功能详解

#### F1 — 基金产品卡片化展示

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 四张卡片展示四类基金，每张卡片显示：基金名称、一句话描述、预期收益区间（合规表述）、风险等级、最低投资额、可投地区、"了解更多"按钮 |
| 交互 | 点击卡片展开详情抽屉页，不跳转新页面 |
| 合规 | 收益区间标注"非承诺收益，仅供参考" |

#### F2 — LP 投资意向预登记表单

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 收集投资人初步兴趣，字段：姓名、机构名称、职位、邮箱、电话、感兴趣的基金类型（多选）、可投地区（多选）、投资规模偏好（单选下拉） |
| 交互 | 提交后显示"感谢"页面 + 24 小时内专人回复承诺 |
| 后端 | 数据存入后台 + 自动发送通知邮件给 Inspira 团队 |

#### F3 — 基金结构可视化解释器

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 使用动态图解（SVG/Canvas）展示资金从 LP → 基金 → 项目的流向，每类基金展示风险收益曲线对比 |
| 交互 | 用户切换四类基金标签，图解自动更新 |
| 设计 | 参考 Bloomberg 级别的金融数据可视化质感 |

#### F4 — 路条信息收集表单

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 开发商提交路条，字段：公司名称、联系人、邮箱、电话、地区（下拉）、项目类型（光伏/储能/风电）、容量（MW）、当前阶段（路条/可研/核准/在建）、预计投建时间、补充说明、文件上传 |
| 交互 | 3 步渐进式表单，每步不超过 3 个字段，降低弃填率 |
| 后端 | 文件上传至加密存储，数据仅内部团队可见 |

#### F5 — 交互式全球市场地图

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 首页 Hero 下方展示可交互地图，标注东南亚、澳洲、中国、欧洲四个区域 |
| 交互 | 鼠标悬停显示区域名称和状态（在谈/意向/已布局），点击进入区域详情 |
| 视觉 | 深色底图 + 渐变光点标注，科技感强 |

#### F6 — 团队背书页

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 核心团队成员卡片，每人包含：照片、姓名、职位、从业年限、代表项目/资产规模、个人简介 |
| 设计 | 避免模板化，使用真实照片和具体数字增强可信度 |

#### F7 — 中英双语切换

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 右上角语言切换按钮，一键切换中文/英文，切换后保持当前页面 |
| 实现 | i18n 国际化方案，内容文件分离管理 |

#### F8 — 市场洞察报告订阅

| 属性 | 说明 |
|------|------|
| 优先级 | P0 |
| 描述 | 页脚和 Insights 页面设置邮件订阅入口，访客输入邮箱即可订阅 |
| 后端 | 对接邮件服务（Mailchimp / SendGrid），定期发送季度报告 |
| 交互 | 订阅成功提示 + 确认邮件 |

### 7.3 技术方案

| 维度 | 推荐方案 |
|------|---------|
| 前端框架 | Next.js（SSR 支持 SEO + 高性能） |
| 样式方案 | Tailwind CSS（快速开发 + 响应式） |
| 地图组件 | Mapbox GL 或 Leaflet（交互式地图） |
| 图表可视化 | D3.js 或 Chart.js（基金结构可视化） |
| 表单后端 | 初期：Supabase / Firebase 简化存储；后期：自建 Node.js 后端 |
| 邮件服务 | SendGrid 或 Mailgun（交易邮件 + 订阅） |
| 国际化 | next-i18next |
| 部署 | Vercel（一键部署 + 全球 CDN） |
| 分析 | Google Analytics 4 + Hotjar（行为分析） |

### 7.4 关键假设

| 编号 | 假设 | 置信度 | 验证方式 |
|------|------|-------|---------|
| A1 | 投资人愿意在官网留下投资意向 | 中 | 上线后观察转化率，如低于 1% 需调整 |
| A2 | 开发商愿意在线提交路条信息 | 中 | 同步线下验证开发商对在线提交的接受度 |
| A3 | 基金结构可视化能降低理解门槛 | 高 | 路演时对比有无可视化时的客户反馈 |
| A4 | 市场地图能增强品牌可信度 | 高 | 通过 Hotjar 观察地图交互热区 |
